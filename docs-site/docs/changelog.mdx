# Changelog

All notable changes to **`@ackplus/mui-tanstack-data-grid`**. This project follows
[Semantic Versioning](https://semver.org/) and the [Keep a Changelog](https://keepachangelog.com/)
format: each release groups changes as **Added** (new features), **Changed** (behaviour/build), **Fixed**
(bug fixes), and **Docs**.

> Upgrading is safe within a major version — `1.x` releases are backward compatible. Install a specific
> version with `npm install @ackplus/mui-tanstack-data-grid@<version>`.

---

## [1.20.4](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.20.4) — 2026-07-11

Backward compatible.

### Fixed
- **`selectOnRowClick` now works.** The prop was declared in the public types but never wired up — a dead
  prop (setting it did nothing). It now toggles a row's selection when you click **anywhere in the row**
  (needs `enableRowSelection`; respects single/multi mode and per-row `isRowSelectable`). Clicks on the
  checkbox, row expander, action buttons, or an editing cell don't trigger it (they stop propagation), and
  it **composes** with `onRowClick` — both run when both are set.

## [1.20.3](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.20.3) — 2026-07-11

Backward compatible.

### Fixed
- **Clicking a row's selection checkbox or expand toggle no longer also fires `onRowClick`.** The
  auto-generated **selection** column (the row checkbox **and** the select-all header checkbox) and the
  **row-expansion** button didn't stop click propagation — so on a grid with both row selection/expansion
  and an `onRowClick` (e.g. one that navigates to a detail page), ticking the box or expanding a row also
  triggered the row click. They now `stopPropagation` (matching the actions cell): the checkbox still
  toggles selection, the button still expands, and clicking a **normal** cell still fires `onRowClick`.

## [1.20.2](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.20.2) — 2026-07-11

Backward compatible.

### Fixed
- **Column resize no longer jumps on the first drag.** Under `fitToScreen`, an untouched column is
  flex-grown **wider** than its base `size`; the instant a resize wrote `columnSizing`, the grow switched
  off and the width snapped back to that base — a visible leftward jump before the drag caught up. The
  resize now **seeds the column's current rendered width at mouse-down** (so it starts from what you see),
  turning the flex→fixed switch into a no-op — resizing is smooth from the first pixel and neighbouring
  columns keep their width. The native drag math, `onChange`/`onEnd` modes, touch, and RTL are unchanged.

## [1.20.1](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.20.1) — 2026-07-11

Backward compatible.

### Fixed
- **`footerFilter` now renders.** The prop was declared on `DataTableProps` (and documented) but the render
  layer never read it — a **dead prop** since it was introduced. It now renders on the **left of the footer**,
  opposite pagination (mirroring how `extraFilter` sits on the right of the toolbar), so a "show deleted"
  soft-delete toggle or any custom footer control appears as intended. The footer now shows whenever
  **`footerFilter`** *or* **`enablePagination`** is set (previously footer chrome required pagination).

### Docs
- **Documentation site rebuilt in the MUI-X layout.** The nav is now grouped into labelled sections —
  **Main features · Server-side data · Components · Customization · Resources** — with **New** and
  **Coming soon** badges. New pages: **Cells, Virtualization, Usage, Quick Filter, Styling recipes,
  Overlays, Custom subcomponents, Events, State, Performance**, plus roadmap stubs (Row grouping, Pivoting,
  Columns/Filter Panel). The props reference gained a reading guide (type = possible values · default ·
  use-case) and richer per-prop notes. Also fixed a site-wide bug where admonition callouts with a custom
  title rendered as literal `:::` text.

## [1.20.0](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.20.0) — 2026-07-10

Backward compatible.

### Added
- **Labelled toolbar (`toolbarVariant`).** Toolbar controls default to compact **icon-only** buttons
  (`'icon'`, with tooltips). Set **`toolbarVariant="text"`** to render each as an **icon + label** button —
  clearer for users who don't recognise an icon at a glance. Applies to every built-in control (search,
  filter, columns, density, export, refresh, reset, saved views, list-view toggle), with the filter count
  and saved-views "unsaved" dot preserved in both variants. All controls now share one **`ToolbarButton`**
  (standard MUI `IconButton`/`Button`), so `slots`/`slotProps` overrides work the same way in either variant.

### Changed
- **The bulk-actions bar is now a contextual overlay** (Material "contextual app bar" pattern). When a
  **toolbar** is shown, the bar is positioned **absolutely over it** — edge-to-edge, the same height — so
  selecting rows causes **zero** height change in any mode (the earlier fix stopped it exceeding `maxHeight`,
  but a non-full grid still grew up to the cap; now it doesn't grow at all). When there's **no toolbar**, the
  bar renders in flow **above the header** instead — so it never covers the column headers or the select-all
  checkbox (matching MUI X / AG Grid, which always keep the header visible). Custom
  `slots.bulkActionsToolbar` components get the same treatment.

### Docs
- The **Features** nav is grouped into **Data · Columns · Rows · Display & layout · Accessibility & i18n**
  (was one long flat list), and the **Saved views** page — which existed but was never linked — is now in the
  sidebar.

## [1.19.0](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.19.0) — 2026-07-10

Adds **List view** — render rows as full-width list items instead of a columnar table, great for narrow /
mobile layouts (a **Pro-only** feature in MUI X, free here) — and fixes a `maxHeight` scroll bug. Backward
compatible.

### Fixed
- **`maxHeight` no longer double-scrolls when a selection bar appears.** The height bound now sits on the
  **root** (not the scroll viewport), so all chrome — toolbar, the bulk-actions bar, footer — lives inside
  the budget and the body flexes. Selecting rows shrinks the body instead of growing the whole grid (and it
  grows back on clear — no fixed reduced height, no empty space, no second scrollbar). Matches how MUI X /
  AG Grid bound the container and flex the body. `height` / `100%` modes were already correct.

### Added
- **List view.** Set `listView` + `renderListItem` to render each row as one full-width item you fully
  control (avatar, text, actions, …). The engine is unchanged — **sorting, filtering, pagination, selection,
  saved views, and virtualization all keep working**; only the row rendering switches.
  - **`renderListItem({ row, rowNode, api, index, isSelected })`** owns the row content.
  - **`enableListView`** adds a built-in **grid ⇄ list toggle** to the toolbar; control it yourself with
    `listView` + `onListViewChange`. The classic responsive pattern is `listView={isMobile}`.
  - New **`rowHeight`** prop (fixed px, overrides the density row height).
  - Themeable: `styleOverrides.listItem`, `slots.listItem` / `slotProps.listItem`, plus
    `slots.gridViewIcon` / `slots.listViewIcon` / `slots.viewModeToggle` for the toggle.
  - Column-only concepts (headers, resize / reorder / pinning, grouped headers, the aggregation footer) turn
    off in list view; sort / filter run through the toolbar and `api`. See [List view](/features/list-view).

## [1.18.3](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.18.3) — 2026-07-09

Deeper MUI-theme integration for **typography and spacing**, plus roomier cells. No API changes; the grid now
follows a custom theme's fonts and spacing.

### Changed
- **Typography follows the theme.** Data cells use `theme.typography.body2` and header / aggregation cells use
  `theme.typography.subtitle2` (family, weight, line-height, letter-spacing) — the hardcoded `fontSize: 0.8125rem`
  / `fontWeight: 600` are gone. Font **size** stays density-scaled via the `--dt-font-size` token, which now
  derives from `theme.typography.body2.fontSize`. Applied directly on the styled cell/header primitives — **no
  extra `<Typography>` wrappers**. *Side effect: the header font now scales with density (it was previously
  pinned at the compact size).*
- **Spacing follows `theme.spacing()`.** Density presets express cell padding — and the toolbar / footer gaps —
  in spacing **units**, so `--dt-cell-padding-*` and the built-in gaps track a custom `theme.spacing`.
- **Roomier cells with vertical padding.** Cells gained density-based **top/bottom** padding, and the padding
  was retuned to be near-square and more generous (default theme: compact ≈ 8/6px, standard 16px, comfortable
  20px, horizontal × vertical). Wrapped or tall content now gets proper breathing room and the row grows with
  it instead of the text touching the row edges. `rowHeight` stays a **min** floor, so ordinary single-line
  rows keep a consistent density height.

### Notes
- The `--dt-*` tokens remain the fast per-instance / `sx` override channel — they just **default** from
  `theme.spacing` / `theme.typography` now instead of hardcoded values. Colours and radius were already
  theme-derived. The div render layer is unchanged: it is 18 `styled('div', { name, slot })` MUI primitives
  (themeable via `styleOverrides`), not raw divs — only three intentional 0-height spacers remain plain.

## [1.18.1](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.18.1) — 2026-07-08

Column-header/separator polish and a clearer, safer toolbar **Reset**. Builds on 1.18.0.

### Fixed
- **Column separators are a single divider (no more double line).** Removed the header cell's box-shadow
  separator that doubled up with the resize handle. Each resizable column now shows **one** full-height
  divider at its trailing edge — subtle (border colour) at rest, accented and thicker on hover / while
  resizing — and it's still the resize grab area (double-click auto-fits).
- **Pinned columns no longer show a broken "dashed" separator.** With the header box-shadow gone, a pinned
  column (e.g. the selection / checkbox column) reads cleanly with just its soft pinned-edge shadow instead
  of a header sharp line that didn't line up with the body.

### Changed
- **Roomier cells.** Standard (medium) density cell padding is now **16px** (was 12px) on header and body
  cells, matching the MUI reference spacing. Compact (8px) and comfortable (16px) are unchanged.
- **"Reset" is now "Reset layout" and resets layout only.** The reset button restores the **column layout**
  (`columnOrder` + `columnPinning` + `columnSizing` by default) and **no longer** wipes filters, sorting, or
  pagination, nor reloads the data — a behaviour change from 1.18.0. Widen it with the new **`resetActions`**
  prop (add `'columnVisibility'` / `'rowPinning'` / `'filters'` / `'sorting'` / `'pagination'`), or call it
  imperatively via **`api.layout.reset(actions?)`**. The old "reset everything + reload" is still available
  as `api.layout.resetAll()`. The refresh button is relabelled **"Refresh data"** and continues to re-fetch
  data only (calls `onFetchData` in server mode).

### Docs
- [Props → Toolbar](/api/props) and [Toolbar](/features/toolbar) document `resetActions` and the layout-only
  Reset semantics.

## [1.18.0](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.18.0) — 2026-07-08

Three things: makes the `slots` / `slotProps` / `sx` customization surface **honest** (declared parts that
the render layer never read now genuinely apply), polishes the default column header and toolbar to match
the MUI DataGrid reference, and adds support for **MUI 8 & 9**. Backward compatible — MUI 5/6/7 keep working
unchanged.

### Compatibility
- **MUI 8 & 9 support.** The `@mui/material` / `@mui/icons-material` peer range now spans
  `^5 || ^6 || ^7 || ^8 || ^9`. The one icon that MUI 8/9 dropped — `@mui/icons-material/DeleteOutline`
  (the standalone outline variant), used by the saved-views delete button — is replaced with the standard
  `DeleteOutlined` theme variant (visually identical, present in every `@mui/icons-material` v5–v9). This
  fixes the Vite build error `Failed to resolve import @mui/icons-material/DeleteOutline` on MUI 9. Verified
  end-to-end against MUI 9.2.0 (install, icon resolution, SSR render, and a strict `tsc` pass of the grid's
  component API).

### Fixed
- **Root `sx` and `slotProps.root` were never applied.** The root rendered with only `className`, silently
  dropping `sx`, `slotProps.root`, and even the caller's own `className` when combined. All three now apply
  and **merge** — the `--dt-*` design tokens and scroll-layout height can no longer be clobbered by an
  injected `style`, and nested `sx` arrays flatten correctly.
- **~20 dead `slots` / `slotProps` keys are now wired.** `scroller`, `grid`, `header`, `headerRow`,
  `headerCell`, `body`, `row`, `cell`, `detailPanel`, `footer`, `loadingOverlay`, `noRowsOverlay`, and every
  toolbar control (`searchInput`, `columnFilterControl`, `columnVisibilityControl`, `densityControl`,
  `viewsControl`, `exportButton`, `refreshButton`, `resetButton`, plus the `toolbar` container) are read by
  the render layer. Structural parts spread your props under their required roles/refs/keyboard wiring (never
  clobbered); leaf controls take your props last (overrides win).
- **Resize handle is no longer a full-height bar.** The column resize divider is now a short, vertically
  centered line (like MUI's column separator) — invisible at rest, revealed on header hover, and highlighted
  while you hover the handle or actively resize. The full-height area remains grabbable (and double-click
  still auto-fits).

### Changed
- **Column header layout matches MUI.** The title and sort icon now grow to fill the cell, so the per-column
  **⋮ menu is pinned to the trailing (right) edge** instead of sitting next to the label. Alignment
  (`left` / `center` / `right`) is preserved.
- **Sort affordance on hover.** Sortable columns now show a faint sort arrow when you hover the header
  (previously the arrow appeared only once a column was sorted), so it's discoverable that the column sorts.
  A sorted column still shows its solid up/down arrow.
- **Toolbar is left-aligned.** All built-in controls now sit together on the leading edge in a stable order
  — search, saved views, filter, columns, density, export, then refresh / reset — mirroring the MUI DataGrid
  toolbar. Only the caller-provided `extraFilter` slot stays on the trailing edge.
- The no-rows overlay's emitted class is now `MuiTanstackDataGrid-noRowsOverlay` (was `-overlay`), matching
  the documented `styleOverrides.noRowsOverlay` key. `loadingOverlay`, `grid`, and `pagination` gained their
  own `styleOverrides` slots.
- `slots.columnPinningControl` is deprecated — it was never read. Column pinning lives in the Columns panel
  (`columnVisibilityControl`) and the per-column header ⋮ menu; drive it programmatically via
  `apiRef.current.columnPinning`.

### Docs
- [Theming → Per-instance overrides](/theming) and [Props → Slots & slotProps](/api/props) now show a
  working `sx` + `slotProps` example and document the merge semantics (yours wins per-property; required
  behavior props are protected).

## [1.17.0](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.17.0) — 2026-06-21

Editing gains whole-row mode and custom editors. Backward compatible (`editMode` defaults to `'cell'`).

### Added
- **Whole-row edit mode.** Set `editMode="row"` and a row's editable cells open **together** with explicit
  **Save** / **Cancel** in the actions column (auto-added — no wiring needed); `processRowUpdate` fires
  **once** per row with the fully-updated row. Start via double-click / Enter / `apiRef.current.editing.startRowEdit(id)`;
  `onRowEditStart` / `onRowEditStop` callbacks; the actions are also exported as `createRowEditAction` to mix
  with your own `getRowActions`.
- **Custom cell editors.** `columnDef.editComponent` now actually drives the inline editor (it was previously
  dead). It receives `DataTableEditComponentProps` (`value` / `onChange` / `onCommit` / `onCancel` / `row` /
  `column` / `align` / `editMode`) and works in both cell and row mode.

### Changed
- `columnDef.editComponent` is no longer a fallback for the column **filter** input — use `filterComponent`
  for that. (No known consumers relied on the fallback.)

### Docs
- [Editing](/features/editing) documents row mode, the `editComponent` editor, and `apiRef.current.editing`.

## [1.16.0](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.16.0) — 2026-06-21

Last remaining backlog slice — two rendering-correctness fixes. Backward compatible.

### Added
- **Right-to-left (RTL).** Set `direction: 'rtl'` on your theme and the grid mirrors itself — pinned
  columns flip side, the resize handle and sort/⋮-menu move to the logical edge, text aligns to the start,
  and tree indentation + column separators mirror. The grid sets `dir="rtl"` on its root, so it works with
  no extra setup. See [Theming → RTL](/theming#6-right-to-left-rtl).

### Fixed
- **Grouped (multi-level) headers** now align correctly: a group header spans its leaf columns' combined
  width and lines up with the leaf row and body cells (previously they could drift under `fitToScreen`).
  Group headers carry no sort/resize/⋮-menu, only the leaf header row takes keyboard focus, and
  `aria-rowindex`/`aria-rowcount`/`aria-colspan` account for the extra header rows. See
  [Columns → Grouped headers](/features/columns#grouped-headers).

### Docs
- New Grouped headers ([Columns](/features/columns)) and RTL ([Theming](/theming)) sections + live demos.

## [1.13.0](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.13.0) — 2026-06-21

First of the remaining backlog slices. Backward compatible.

### Added
- **Column header menu.** Each column header gains a quiet **⋮** menu (revealed on hover or keyboard

## [1.10.0](https://www.npmjs.com/package/@ackplus/mui-tanstack-data-grid/v/1.10.0) — 2026-06-20

Completes **Tier C** — keyboard accessibility, inline editing, and tree data. Backward compatible (no API
removed); the grid now uses the ARIA grid roles and keyboard nav is always on.

### Added
- **Keyboard navigation + ARIA grid.** WCAG grid-pattern operability: `role="grid"` (`treegrid` for tree
  data), `aria-rowindex`/`aria-colindex`/`aria-rowcount`/`aria-colcount`, header `aria-sort`. A roving
  tabindex with **↑ ↓ ← →**, **Home/End**, **Ctrl+Home/End**, **PageUp/PageDown** moves the focused cell;
  **Enter/F2** activates it (sort a header, toggle a control, or start editing); focus follows paging and
  scrolls virtualized rows into view. See [Accessibility](/features/accessibility).
- **Inline cell editing.** Mark a column [`editable`](/features/editing); double-click or Enter/F2 to edit.
  Type-aware editor (text / number / date, dropdown for `select`/`boolean`); Enter or blur commits, Escape
  cancels. `processRowUpdate(newRow, oldRow)` persists the change (reverts + `onProcessRowUpdateError` on a
  throw); without it, edits update the grid's local data.
- **Tree data.** [`getSubRows`](/features/tree-data) renders hierarchical rows — an expander only on rows
